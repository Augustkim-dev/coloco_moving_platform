## 1. 제품 개요

### 1.1 제품 한 줄 정의
자연어 입력 또는 **가이드 대화(Rule-based)** 또는 폼 입력을 통해 사용자의 이사 정보를 수집하고, AI가 조건에 맞는 최적의 이사업체를 자동 매칭해주는 모바일 중심 중계 플랫폼

### 1.2 기획 배경
- 이사 견적 서비스는 **입력 정확도**에 따라 품질이 결정됨
- 기존 서비스들은 폼 입력 위주로, 사용자 진입 장벽이 높음
- 여러 업체에 견적을 받아 비교하는 방식은 고객에게 시간적 부담
- 자연어 기반 입력 + **가이드 대화 + 폼** 3가지 입력 모드 + AI 자동 매칭으로 입력 피로도와 의사결정 부담을 동시에 해소

### 1.3 핵심 가치 제안 (Value Proposition)
1. **가이드 대화(기본)**로 버튼/카드 선택만으로 견적 신청 완료 가능 (AI 불필요)
2. **자연어 한 줄 입력**으로도 여러 필드를 한번에 채울 수 있음 (AI 파싱)
3. **팁 카드(일러스트+가이드)**로 이사 경험 부족한 사용자도 쉽게 판단
4. **AI 자동 매칭**으로 조건에 맞는 최적 업체 배정 (견적 비교 X)
5. **모바일에서 한 손 조작** 가능한 UX

### 1.4 참고 서비스 분석

| 서비스 | 모델 | 특징 | 벤치마킹 포인트 |
|--------|------|------|-----------------|
| **아정당** | 견적 비교 중계 | 파트너 업체 배정, 안심바로케어 보상 | 이사 유형 분류, 보상/보험 체계 |
| **이사대학** | 유형별 매칭 | 원룸/가정/용달/사무실/해외 분류, 층수·운반방식 상세 입력 | 입력 필드 구성, 거주형태·평수 옵션 |
| **영구이사** | 본사→지점 자동배정 | 550개 지점, 암행점검, 칭찬후기 시스템 | 지점 관리 체계, 품질관리 |
| **화물맨(cargo-man)** | AI 채팅 + 폼 하이브리드 | Gemini로 대화 파싱 → 실시간 폼 반영 | **핵심 UX 패턴 그대로 차용** |
| **짐싸만** | 가이드 대화형 | 채팅 UI에서 버튼/카드 선택, 팁 카드(일러스트) 제공 | **가이드 대화 + 팁 카드 UX 벤치마킹** |

### 1.5 핵심 차별점 요약

| 항목 | 기존 서비스 | 본 서비스 |
|------|-------------|-----------|
| 입력 방식 | 폼 직접 입력만 | **가이드 대화(기본)** + 자연어 AI 파싱 + 폼 3모드 |
| AI 비용 | — | 가이드 대화로 AI 호출 70~100% 절감 |
| 사용자 가이드 | 텍스트 설명만 | **팁 카드** (일러스트+가이드 텍스트) |
| 매칭 방식 | 여러 업체 견적 비교 | AI 자동 매칭 (최적 1개 배정) |
| 누락 정보 처리 | 폼 validation만 | 가이드 대화 순차 질문 + AI 대화 보완 |
| 짐 정보 | 평수 기반 추정 | 가전·가구·특수짐 개별 파싱 |

## 2. 목표 사용자

### 2.1 고객 (이사 의뢰자)

| 페르소나 | 특징 | 핵심 니즈 |
|----------|------|-----------|
| **1인 가구 사회초년생** (20~30대) | 원룸/오피스텔, 짐 적음, 이사 경험 부족, 이사 용어 미숙 | 간편한 접수, 저렴한 가격, 쉬운 UX |
| **신혼부부/젊은 가족** (30~40대) | 아파트 이사, 가전·가구 多 | 안전한 포장, 신뢰할 수 있는 업체 |
| **기업 담당자** | 사무실 이전, 대규모 | 체계적 일정 관리, 전문 인력 |

**공통 특성:**
- 이사 관련 용어에 익숙하지 않음
- 빠른 입력과 간단한 UX 선호
- 모바일 환경 사용 비중 높음

### 2.2 업체 (이사 서비스 제공자)
- 화물자동차 운송주선사업 허가 보유 업체
- 영업 지역, 전문 분야(원룸/가정/사무실), 보유 차량 등으로 프로필 관리

### 2.3 관리자
- 플랫폼 운영팀: 매칭 모니터링, 업체 관리, 클레임 처리

## 5. AI 입력 해석 설계

### 5.1 파싱 정책

- 자연어 입력은 **JSON 스키마 기반으로 구조화 파싱**
- **명확한 정보**는 즉시 폼에 반영
- **모호하거나 누락된 필수 정보**는 반드시 추가 질문 생성
- **"모름" 허용 필드**는 무리하게 추론하지 않음

### 5.2 파싱 규칙 예시

| 입력 | 파싱 결과 | 비고 |
|------|-----------|------|
| "원룸" | 이사 종류 = 원룸 | 직접 매핑 |
| "용달" | 이사 형태 = 용달이사 | 직접 매핑 |
| "포장이사" | 이사 형태 = 포장이사 | 직접 매핑 |
| "3월 말" | 이사 희망일 = 3/25~31 (추정) | 범위로 처리, 확인 질문 |
| "다음주 토요일" | 이사 희망일 = 2026-02-14 | 정확 매핑 |
| "강남에서 마포로" | 출발지 = 강남구, 도착지 = 마포구 | 구 단위 추출, 상세 주소 질문 |
| "엘베 없어요" | 엘리베이터 = 없음 | 출발/도착 중 맥락 판단 |
| "냉장고랑 세탁기 있어요" | 대형 가전 = [냉장고, 세탁기] | 리스트 추출 |
| "킹사이즈 침대" | 대형 가구 = [침대], 메모: 킹사이즈 | 상세 정보는 메모로 보존 |
| "피아노 있어요" | 특수 짐 = [피아노] | 특수 짐 플래그 |

## 6. 유저 플로우

### 6.1 고객 플로우 (메인)

```
[랜딩 페이지]
    ↓
[견적 신청 페이지] ← 가이드 대화 + 자유 입력 + 폼 3모드
    │
    ├── 모드 A: 가이드 대화 (기본) ⭐ AI 불필요
    │   ├── 시스템이 순서대로 질문 → 버튼/카드/캘린더로 답변
    │   ├── 팁 카드(일러스트)로 판단 어려운 항목 가이드
    │   ├── 각 답변에 "수정" 버튼 → 이전 답변 수정 가능
    │   └── 모든 필수 항목 완료 시 → 정보 확인 페이지
    │
    ├── 모드 B: 자유 입력 (선택적 전환) → AI 파싱
    │   ├── 가이드 진행 중 텍스트 입력창 탭 → 자유 입력 모드
    │   ├── 자연어 입력 → Gemini 파싱 → 여러 필드 한번에 채움
    │   └── 파싱 후 남은 미입력 필드로 가이드 대화 재개
    │
    ├── 모드 C: 폼 직접 입력 (탭 전환)
    │   ├── [폼] 탭으로 전환 → 전통 폼에서 직접 수정
    │   └── 가이드 대화 답변이 실시간으로 폼에 반영
    │
    ↓
[정보 확인 & 동의] ← 전체 입력 요약 + 개인정보 동의 + 이사화물 표준약관
    ↓
[견적 요청 전송]
    ↓
[AI 매칭 진행] → 조건 기반 최적 업체 자동 배정
    ↓
[매칭 결과 안내] → 배정 업체 정보 + 예상 비용 범위
    ↓
[업체 연락] → 업체에서 고객에게 직접 연락
    ↓
[방문견적 / 계약 확정]
    ↓
[이사 완료]
    ↓
[리뷰 작성] → 별점 + 한줄평
```

### 6.2 업체 플로우

```
[업체 회원가입] → 사업자 등록증, 허가증 제출
    ↓
[프로필 등록] → 서비스 지역, 전문 분야, 보유 차량, 보험 정보
    ↓
[심사 승인] → 관리자 확인 후 활성화
    ↓
[매칭 알림 수신] → 조건에 맞는 이사 건 Push/SMS
    ↓
[이사 건 상세 확인] → 고객 입력 정보 전체 열람
    ↓
[수락 / 거절]
    ↓
[고객 연락] → 방문견적 일정 조율
    ↓
[이사 완료 보고]
    ↓
[고객 리뷰 수신]
```

### 6.3 관리자 플로우

```
[대시보드] → 실시간 매칭 현황, 일별/주별 통계
    ↓
[매칭 관리] → 수동 재배정, 매칭 실패 건 처리
    ↓
[업체 관리] → 업체 등록 심사, 평점 관리, 정지/해제
    ↓
[고객 관리] → 신청 이력, 클레임 처리
    ↓
[정산 관리] → 업체별 수수료 정산
```

## 7. AI 매칭 로직

### 7.1 매칭 기준 (우선순위)

| 순위 | 기준 | 가중치 | 설명 |
|------|------|--------|------|
| 1 | **지역** | 30% | 출발지/도착지 커버 가능 여부 |
| 2 | **이사 형태** | 25% | 업체의 전문 분야 일치 |
| 3 | **일정** | 20% | 희망일 가용 여부 |
| 4 | **차량** | 15% | 필요 톤수 보유 여부 |
| 5 | **평점** | 10% | 고객 리뷰 기반 평점 |

### 7.2 매칭 프로세스

```
[견적 요청 접수]
    ↓
[필터링] → 지역 + 이사 형태 + 일정으로 후보군 추출
    ↓
[스코어링] → 가중치 기반 점수 산출
    ↓
[최적 업체 1곳 배정]
    ↓
[업체 수락 대기] → 30분 이내 미응답 시 차순위 업체에 재배정
    ↓
[거절 or 타임아웃] → 자동 재배정 (최대 3회)
    ↓
[3회 실패] → 관리자 수동 개입 알림
```

### 7.3 초기 업체 수 부족 시 대응 (MVP)

- 초기에는 **규칙 기반 매칭** (지역 + 이사 형태 + 일정 필터링만)
- 업체 수 < 3개 지역은 **수동 매칭** (관리자 개입)
- 업체 풀 확보 후 스코어링 알고리즘 고도화

## 9. UX/UI 설계 원칙

### 9.1 모바일 퍼스트

| 원칙 | 구현 방안 |
|------|-----------|
| **한 화면 한 질문** | 가이드 대화에서 한 번에 하나의 질문만 표시 |
| **3모드 전환** | 데스크톱: 좌우 분할(채팅+폼) / 모바일: 하단 탭 전환(💬 채팅 / 📋 폼) |
| **가이드 대화 기본** | 모바일 진입 시 가이드 대화 모드 기본 활성 (AI 불필요) |
| **자유 입력 전환** | 입력창 탭 시 자유 텍스트 모드 전환 → AI 파싱 |
| **팁 카드** | 판단 어려운 질문(트럭 대수, 작업인원) 전에 일러스트+가이드 표시 |
| **하단 고정 CTA** | "견적 신청하기" 항상 하단 고정 (sticky) |
| **진행률 표시** | 상단에 폼 완성도 % (필수 필드 기준) |
| **수정 버튼** | 각 답변 버블에 "수정" 버튼 → 이전 답변 재선택 가능 |
| **터치 친화** | 버튼 최소 48px, 카드형 선택, 바텀시트, 여백 충분 |

### 9.2 입력 피로도 최소화

| 원칙 | 구현 방안 |
|------|-----------|
| **선택형 UI 우선** | 가능한 모든 필드를 Select/Toggle로 |
| **숫자 입력 최소화** | 범위 선택형으로 제공 (1~5개, 6~10개 등) |
| **"모름" 옵션** | 모든 선택 필드에 "모름" 제공 |
| **실시간 연동** | 폼 수정 ↔ 채팅 메시지 양방향 동기화 |
| **자동 저장** | 입력 중 이탈해도 데이터 보존 |
