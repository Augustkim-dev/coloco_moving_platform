# 007. 랜딩페이지 위저드 + 차량선택 + 컴팩트 확인페이지

## Context

현재 랜딩페이지는 Google Play 다운로드를 유도하는 마케팅 페이지. 사용자가 웹에서 바로 견적을 시작할 수 없음.
요청: 랜딩페이지에서 주거형태 → 평수 → 이사형태 → 차량선택까지 미리 진행한 후, 가이드 입력 화면으로 넘어가서 이사예정일/주소 등 나머지를 입력받고, 마지막에 컴팩트한 확인 페이지를 보여주는 플로우.

### 결정 사항
- 가이드 입력 첫 질문: **이사 예정일** (Step 1)부터 시작
- /estimate 직접 접속 시: **랜딩으로 리다이렉트** (차량 선택 필수)
- 이사형태: **보관이사 포함 5개** (용달, 일반, 반포장, 포장, 보관)

---

## 작업 범위

### A. 스키마 변경 (`src/types/schema.ts`)

1. `VehicleType` 타입 추가:
   ```typescript
   export type VehicleType = 'damas' | 'labo' | '1ton_cargo' | '1ton_wing' | '1.4ton_cargo' | '1.4ton_wing';
   ```

2. `VEHICLE_TYPE_LABELS` 라벨 매핑 추가

3. `VEHICLE_DATA` 상수 추가 — 각 차량별 정보:
   - name, description, dimensions (가로x세로x높이), image path, startPrice 텍스트
   - 다마스: "16만원부터", 소형 가전 1~2개, (1.4~1.7)*1.2*1m
   - 라보: "18만원부터", 소형 가전 가구, (1.8)*1.3*1.1m
   - 1톤 카고: "21만원부터", (2.4)*1.4*1.2m
   - 1톤 윙바디: "23만원부터", (2.4)*1.4*1.2m (날개형)
   - 1.4톤 카고: "30만원부터", (2.0~2.9)*1.4*1.5m
   - 1.4톤 윙바디: "32만원부터", (2.0~2.9)*1.4*1.5m

4. `Conditions` 인터페이스에 `vehicleType: VehicleType | null` 추가

5. `createDefaultSchema()`에서 `conditions.vehicleType: null` 초기화


## 검증 방법

1. 랜딩페이지 접속 → 주거형태 → 평수 → 이사형태 → 차량 선택 → /estimate 이동 확인
2. /estimate에서 step 1(이사 예정일)부터 시작, step 2/3/4 스킵 확인
3. 가이드 플로우 완료 → 컴팩트 확인 페이지에서 차량 정보 포함 확인
4. /estimate 직접 접속 → 랜딩으로 리다이렉트 확인
5. 모바일/데스크탑 반응형 확인
6. 위저드에서 뒤로가기 동작 확인
